Interesting Time During My Internship (Web Developer)

One of the most interesting time I had during my internship was making API. I had never done something like that before and frankly didn’t know what an API was, sure I knew the full form but not what it actually meant. The projects I had done till then on web development were intended for humans. This was my first time when I had to make an application where the user was another application (in my case an android app).

So when I was asked to make an API, I quickly read up on wikipedia about it and all the stack exchange queries I could find. What I didn’t understand was how to make one. Now I am not talking about the syntax of it, which is relatively easy, all you have to do is parse the data coming on the url and provide the data (serialize the data into JSON).

What I struggled with is understanding what the correct format is for making the API response data. I didn’t know the rules or the standards of an API. So I just made a API response which as expected was really bad since it didn’t cover all the cases of the problem. Like for example if the URL is hit without any data, I had given no response to it and it would just show empty.

I made a lot of mistakes but my senior colleagues were patient with me and gradually I figured out how an API should work and had fun doing it.
